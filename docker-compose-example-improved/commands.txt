1. Собрать и запустить контейнеры:
'-d' - в фоновом режиме

docker-compose up -d

2. Проверить статус контейнеров:

docker ps

3. Остановить контейнеры (если работаем с группой):

docker-compose stop

4. Повторный запуск:

docker-compose up -d

5. Запуск с пересборкой: (если изменили dockerfile, например)

docker-compose up -d --build

6. Снести контейнеры (остановка с удалением):

docker-compose down

7. Получить информацию о контейнере:

можно по id: docker inspect 5109f6cfdf3f
можно по имени: docker inspect postgres_db_instance
найти строку с ip-адресом, но мы можем использовать название сервиса вместо ip-адреса:
docker inspect postgres_db

в корне папки должен быть файл с разрешением .yml - в нем описывается, какие контейнеры и как нужно поднять
'docker-compose' может использовать образы из 'docker-hub', а может из исп-ть образы, которые автоматически билдятся из docker file

'docker compose' описывает, как поднимать контейнеры (не сборку образа), он может ссылаться либо на образы, либо на docker file
